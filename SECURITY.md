# Security Policy

## Supported Versions

Please check the table below to see which versions of **Universal Daily Rewards** are currently receiving security updates. We highly recommend always running the latest version available on our Modrinth or GitHub pages.

| Version | Supported          | Notes |
| ------- | ------------------ | ----- |
| 4.0.x   | :white_check_mark: | Active development & current major release. |
| 3.1.x   | :white_check_mark: | Critical security patches only. |
| < 3.0.x | :x:                | End of life. Please update. |

## Reporting a Vulnerability

Security is a top priority for server-side mods. If you discover any vulnerability (such as command injection, severe NBT exploitation, or crash exploits) in Universal Daily Rewards, **please DO NOT open a public GitHub Issue**. 

Instead, report it privately so we can patch it before it becomes public knowledge and affects other servers.

### How to report:
1. **Discord (Preferred):** Send a Direct Message to `[SrKalopsia]` on Discord (you can find me in the main community server).
2. **Email:** Send a detailed report to `[srkalopsia@email.com]`.

### What to include in your report:
* The version of the mod and the Minecraft server software (Fabric/Quilt, version) you are using.
* A detailed description of the vulnerability.
* Step-by-step instructions to reproduce the issue.

We will acknowledge receipt of your vulnerability report as soon as possible and strive to release a patch promptly. Thank you for helping keep the community safe!
